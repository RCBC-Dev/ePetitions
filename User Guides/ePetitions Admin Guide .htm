Password Policy

The password for the account must meet minimum strength requirements, which uses a library of common passwords and patterns to advise the user on how to make their password more secure. Passwords do not have to contain numbers, uppercase, special characters but do have to be complex enough to meet the standard. In most cases 3 words will meet the password requirements if these words are not in the 100 most common passwords.

Some common passwords will give advice like this:

p@ssw0rd 0/4 this is a common password and replacing a with @ and o with 0 is very common method of trying to make your password less guessable - but as we see about does not help very much to prevent someone guessing a password.

Examples:

If someone is struggling to come up with a password, ask them to pick 3 words they can remember more uncommon words are better but even something like tigerloafbread meets the minimum requirements scoring 3 out of 4.

bunk bed jumping 4/4

perry chases cars 4/4

ePetitionsaregood 4/4

You can see it is not hard to meet the minimum requirements, ensuring the users create safe passwords that are not easy to guess.

The password must meet the minimum score before the account can be created.

This password policy is also enforced when a password reset is requested.

ePetition Approvals and Decline Process

When an administrator logs into the ePetition system it will automatically take the administrator to the ePetitions Approvals page. This page will list any ePetitions that are awaiting approval.

Here we have two ePetitions that were created for test purposes, you can see that their status is Not Approved. If we click on the blue writing which is the title of the ePetition we see the petition details.

 

If we are happy with the petitioner details, and the petition title and contents then we can approve this petition. This will send the petitioner an email and make the petition available to the public to allow other members of the public to sign it electronically.

 

If the petitioner details are not complete, for example: the user is trying to create a petition anonymously, or the petition title and details are not fit to be shared with the public then the ePetition can be declined. This again will send the petitioner an email but will not publish the ePetition.

Click the Remove This Signature button

The signature is removed from the ePetition, there is no way through the front end to re-activate a signature, so please make sure this is something you want to do.

Inactive ePetitions

Users can click on the Inactive ePetitions link which will take them to ePetitions that have passed their end date.

They will be able to see the list, and the details  but can no longer sign these ePetitions because they have expired.

Search ePetitions

Users can search for part of the title of the ePetition, it will return all results that match the search  this will also show inactive ePetitions.  The details can be viewed, and active ePetitions can be signed.

Register an Account.

Users are reminded that they only need an account if they want to create a petition, we take only their email address  so that personal details are not collected, until the email address has been verified.

The user is sent an email address with a link to click on which contains their userID and a secret key that verifies the user account.

The link https://shawn-dev.redclev.net/epetition/validate.php?user=32&key=5o09bnv9MD312411070621SJbwqtSv6F the link takes them to a page that validates the user and key matches.

The user provides their personal details, which are required to identify them  along with phone numbers (not mandatory) and a password for their account.  Once all fields are complete, they can click Create my Account the button is not enabled until a secure password is provided.

Password Policy

The password for the account must meet minimum strength requirements, which uses a library of common passwords and patterns to advise the user on how to make their password more secure.  Passwords do not have to contain numbers, uppercase, special characters  but do have to be complex enough to meet the standard.  In most cases 3 words will meet the password requirements if these words are not in the 100 most common passwords.

Some common passwords will give advice like this:

p@ssw0rd  0/4  this is a common password and replacing a with @ and o with 0 is very common method of trying to make your password less guessable - but as we see about does not help very much to prevent someone guessing a password.

Examples:

If someone is struggling to come up with a password, ask them to pick 3 words they can remember  more uncommon words are better but even something like tigerloafbread meets the minimum requirements scoring 3 out of 4.

bunk bed jumping 4/4

perry chases cars  4/4

ePetitionsaregood  4/4

You can see its not hard to meet the minimum requirements, ensuring the users create safe passwords that are not easy to guess.

The password must meet the minimum score before the account can be created.

This password policy is also enforced when a password reset is requested.

Creating a new ePetition

Once a user has logged in successfully, they will see this screen.

This page My ePetitions is the users home page, and will display any petitions they have created  along with a button allowing them to create a new ePetition.

Clicking on Create a new ePetition button take them here:

The petition details including the Petition Title are mandatory fields with minimum 6 characters per field, and maximum of 100 characters for the Title and 1000 characters for the Details.

The user also must select the length of time that the petition runs for.

When they have completed this form, they can submit the ePetition for approval by Redcar and Cleveland staff in charge of ePetitions.

When submitted:

Now this petition will appear on the users home page  along with its start and end date  and the number of signatures and status.

A new ePetition automatically adds the petitioner to the ePetition  so it will always have at least 1 signature.

Viewing my ePetitions

This is the default view for a logged in user:

It will show a list of the users ePetitions, and the Start, End Dates and number of signatures and the status as above.

Users can create as many ePetitions as they want  they will not be published until they have been approved.

Logged in users can only see their own petitions and create new petitions, the ability to view and sign other petitions is disabled, the user must log out to view or sign other petitions.

Logging Out

A logged in user simply clicks on the Logout button from the navigation bar.

ePetition Declined

If the ePetition is not appropriate, contains bad language or content that should not be published or is not appropriate, i.e., it relates to a Planning issue  then the ePetition will be declined.  The user will receive an email notifying them of this, along with the reason/cause of the decline.

The email will contain the title of the ePetition, and gives the reason for the decline  with some free text comments from the administrator.

There is the option to call the administrator to discuss why this course of action was taken.

The user will also see that the status of their ePetition has been changed to Declined.
